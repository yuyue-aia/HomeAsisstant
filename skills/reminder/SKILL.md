# 提醒 Skill

对应工具：`manage_reminder`（action: create / list / cancel）。配套：`get_current_time`。

## 操作规则

### create —— 创建提醒

- 用户说"提醒我..."、"到点喊我..."→ `action=create`。
- **时间解析由你来做**：先调 `get_current_time` 取当前本机时间，再把"明天下午4点""10 分钟后""下周一上午 9 点"换算成本机时区的 ISO 8601 字符串（带 `+08:00` 偏移），作为 `fire_at_iso` 传入。
- 重复规则：没说重复 → 默认 `once`；用户说"每天/每日"→ `recurrence=daily`。

#### 提前量判断（关键 UX 点）

| 事件类型 | 处理 | 示例 |
|---|---|---|
| 准备/出门/接送/会议/上课/坐车/赴约 | **自动提前 15 分钟**，`text` 写成"再过十五分钟…" | "明天下午 4 点送余跃去打球"→ `fire_at_iso=15:45`，`text="再过十五分钟送余跃去打球"` |
| 吃药/喝水/起床/睡觉/打卡/烧水好了/饭好了/番茄钟到时间 | **不提前**，按点触发 | `fire_at_iso` 就是用户说的时间，`text` 就是事件本身 |
| 用户明确说了提前量（"提前半小时叫我""提前 5 分钟""到点叫我"） | **严格按用户说的来**，不要自作主张 | — |
| 拿不准 | 按"按点触发"处理，**不要乱加提前量** | — |

### list —— 查看提醒

用户说"我有什么提醒""今天还有什么事"→ `action=list`。

### cancel —— 取消提醒

用户说"取消那个打球的提醒"→ `action=cancel`，把"打球"作为 `query` 传入。

## 播报规则

工具返回的 `message` 只是给你的**语义提示**（如"已创建提醒"），**不要原样念**。

你要根据 `reminder/items/matches` 里的 `nextFireAtIso` + `recurrence` + `text`，结合当前时间，自己组织成自然中文播报。

### 创建确认

按用户说的**原始事件时间**播报，不是 `fire_at_iso`：

- 用户说"明天下午 4 点送余跃去打球"，确认应念："好的，明天下午四点提醒你送余跃去打球。"
- **不要**说"三点四十五"——那是底层触发时间，会让用户混乱。

### 时间表达模板

- `once`，今天/明天/后天 → 用相对词："好的，明天下午四点提醒你送余跃去打球。"
- `once`，更远 → 用月日："好的，五月三十号上午九点提醒你开会。"
- `daily` → 用"每天"+ 时间："好的，每天早上七点提醒你吃药。"

### list 多条

- 按 `nextFireAtIso` 升序念。
- 最多念 3 条，多了说"还有 X 条"。
- 念的时候参考 `text`——它已经包含了"再过十五分钟…"这种自然措辞，**直接用即可**。

### cancel ambiguous

把每条候选用同样的自然中文念出来，反问"你要取消哪一个？"。

## 口播细则

- 分钟为 0 时省略不念。
- 30 分可以念"半"。
- 用 **上午 / 中午 / 下午 / 晚上**，而不是 24 小时制。
