# 游戏管理 Skill

负责小朋友的游戏机时长管理，对应工具：`control_game_console`。

## 适用场景

用户（通常是小朋友）说出以下意图时启用：
- "我想玩游戏" / "给我开游戏" / "可以玩游戏吗"
- "我玩多久了" / "还能玩多久" / "现在谁在玩"
- "我不玩了" / "停一下" / "下机"

## 操作规则

### 必须先确认是谁

- 只能给 **余晓** 或 **余跃** 开游戏，不要凭声音猜。
- 上来就用一句话问："是余晓还是余跃想玩？"
- 其他人（爸爸妈妈、奶奶、客人……）一律礼貌拒绝，不调用工具。

### 三类请求 → 三种 action

| 用户意图 | action | 关键参数 |
|---|---|---|
| 想开始玩 | `start_game` | `who`、`minutes`（5~60）|
| 想停下 | `stop_game` | `who`（如能确认）|
| 查状态 / 还能玩多久 / 谁在玩 | `status` | — |

### start_game 必须先问清分钟数

- 不要替小朋友决定时长，先问："你想玩多少分钟？"
- 范围 5~60 分钟。小朋友说"一直玩""玩到吃饭"这种没数字的，引导给一个具体数字。
- 听到"想玩游戏"时**先调 `status`** 看一眼今日配额：如果今日已无剩余，直接告诉小朋友"今天玩完了，明天再来"，**不要再问分钟数**。

### 播报方式

- 工具返回的 `message` 字段已经是面向小朋友的话术，**原样念出**即可。
- 不要复述 `reason` 字段（那是给系统看的）。
- 拒绝其他人时温柔点："这个游戏机只能给余晓和余跃开哦。"
