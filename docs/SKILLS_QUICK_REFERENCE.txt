┌──────────────────────────────────────────────────────────────────────────────┐
│ 1. HOW TO ADD A NEW SKILL                                                   │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ Step 1: Create directory                                                    │
│   mkdir skills/my-new-skill                                                │
│                                                                              │
│ Step 2: Create SKILL.md file with frontmatter                              │
│   cat > skills/my-new-skill/SKILL.md <<EOF                                 │
│   ---                                                                        │
│   name: my-new-skill                                                        │
│   description: Brief one-sentence description of what this skill does      │
│   ---                                                                        │
│                                                                              │
│   # My New Skill                                                            │
│                                                                              │
│   ## How to use                                                             │
│   Detailed instructions, tool mappings, decision paths, etc.                │
│   EOF                                                                        │
│                                                                              │
│ Step 3: Restart agent (required for discovery)                             │
│   Skills are discovered ONLY at startup via discoverSkills()               │
│   NO hot-reload during runtime                                             │
│                                                                              │
│ Step 4: Verify discovery                                                    │
│   Check logs for: "skills.discover.done" with your skill listed           │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────┐
│ 5. RUNTIME SKILL LOADING                                                    │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ Trigger:  LLM judges user request matches a skill description             │
│           Example: user says "放一首歌" → matches "music" description     │
│                                                                              │
│ Action:   LLM calls: load_skill(name: "music")                            │
│           Tool: src/agent/tools/load-skill.tool.ts                        │
│                                                                              │
│ Handler:  createLoadSkillTool() callback                                   │
│           Calls: loadSkillBody(skills, "music")                           │
│           Reads: skills/music/SKILL.md (full content, no frontmatter)     │
│           Returns: { ok: true, name, directory, body }                    │
│                                                                              │
│ Result:   LLM now has:                                                     │
│           - Detailed skill instructions in context                         │
│           - Access to tools (search_music, control_music_player, etc.)    │
│           - Can now make informed tool calls                               │
│                                                                              │
│ Caching:  NONE - bodies read from disk each time (memory efficient)       │
│           Metadata (name, description) cached in memory                    │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

│                                                                              │

│   Default: ./skills                                                        │
│   Format:  Colon-separated paths (multiple skill directories)              │

┌──────────────────────────────────────────────────────────────────────────────┐
│ 7. SKILL METADATA FORMAT (SKILL.md frontmatter)                             │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ Required Fields (exactly 2):                                               │
│                                                                              │
│   name: <string>                                                           │
│   - Unique skill identifier                                               │
│   - Must match directory name (e.g., skills/music/ → name: music)        │
│   - Used to look up skill with load_skill tool                           │
│   - Case-insensitive matching when loading                               │
│                                                                              │
│   description: <string>                                                    │
│   - One-sentence description for LLM decision-making                       │
│   - Example: "处理'放/来首/想听/...'等点歌与音乐控制请求"                  │
│   - Appears in 【可用技能】 section of system prompt                      │
│   - Helps LLM decide when to call load_skill                             │
│                                                                              │
│ Format Rules:                                                              │
│   - YAML syntax (simple key: value format)                                │
│   - Enclosed by --- delimiters                                            │
│   - No third-party parser (custom simple parser used)                     │
│   - Values can be quoted: name: "my-skill" or name: my-skill              │
│   - Comments allowed (lines starting with #)                              │
│                                                                              │
│ Example:                                                                   │
│   ---                                                                       │
│   name: music                                                              │
│   description: 处理点歌与音乐控制请求                                      │
│   ---                                                                       │
│                                                                              │
│ Body (Markdown):                                                           │
│   Everything after the closing --- is treated as skill instructions        │
│   Can include:                                                             │
│   - Tool mappings                                                          │
│   - Decision paths / logic tables                                          │
│   - Implementation guidelines                                              │
│   - Examples                                                               │
│   - References to helper files in skill directory                          │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────┐
│ 8. IMPORTANT TIMING CONSTRAINTS                                             │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ Startup (T=0):                                                             │
│   ✓ Skills discovered via discoverSkills() (ONE-TIME ONLY)                │
│   ✓ Metadata cached in memory                                             │
│   ✓ System prompt built with skill list                                   │
│   ✓ Agent ready                                                            │
│                                                                              │
│ Runtime (T>0):                                                             │
│   ✓ Skill descriptions available to LLM in system prompt                  │
│   ✓ LLM can call load_skill() to fetch full instructions                  │
│   ✓ SKILL.md body read from disk on-demand                                │
│   ✗ New SKILL.md files NOT discovered (no hot-reload)                     │
│   ✗ Edits to SKILL.md NOT reflected (stale reads possible)                │
│                                                                              │
│ How to Reload:                                                             │
│   → Restart the agent/application                                         │
│   → Triggers new OpenAIAgentRuntime() constructor                         │
│   → Calls discoverSkills() again                                          │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────┐
│ 10. COMMON MISTAKES & GOTCHAS                                               │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ ❌ Adding SKILL.md without restarting                                      │
│    → New skills won't be discovered                                        │
│    ✓ Fix: Restart the application                                         │
│                                                                              │
│ ❌ Typo in frontmatter field names                                         │
│    → Example: "skill_name:" instead of "name:"                            │
│    → Example: "desc:" instead of "description:"                           │
│    ✓ Fix: Check log "skills.discover.missing_frontmatter"                 │
│                                                                              │
│ ❌ Directory name doesn't match skill name                                 │
│    → Example: skills/music-player/ with name: music inside               │
│    ✓ Fix: Make directory name = skill name (usually)                      │
│           or update name: field to match                                    │
│                                                                              │
│ ❌ Editing SKILL.md and expecting changes without restart                  │
│    → Metadata re-scanned at startup only                                  │
│    → Bodies read fresh from disk on load_skill call (this one works)     │
│    ✓ Fix: Restart agent to update metadata                                │
│           Body edits work at runtime (skill must be loaded first)          │
│                                                                              │
│ ❌ LLM not calling load_skill even though skill is needed                 │
│    → Check that skill description is in system prompt                     │
│    → Description might be too unclear/specific                            │
│    ✓ Fix: Review description, consider making it more general             │
│           Add a comment in SKILL.md asking LLM to load it                │
│                                                                              │
│ ❌ Case sensitivity issues with skill names                                │
│    → load_skill matching is case-insensitive (normalized)                │
│    ✓ Usually not an issue, but be consistent in descriptions              │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
