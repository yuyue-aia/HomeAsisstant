    │  │
    │  ├─ readFileSync("skills/music/SKILL.md")
    │  │
    │  ├─ parseFrontmatter(raw)  [Line 178]
    │  │ └─ Extract body only (frontmatter removed)
    │  │
    │  └─ Return: {
    │        ok: true,
    │        name: "music",
    │        directory: "skills/music",

    │     }
    │
    └─ Tool returns result to LLM

    ▼

    │
    ├─ Now sees full SKILL.md body:

MEMORY & CONTEXT USAGE PATTERN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

System Prompt at Runtime:
┌─────────────────────────────────────────────────────┐
│ BASE_INSTRUCTIONS (fixed)                           │
│ - Output format rules                              │
│ - Tool usage guidelines                             │
│ - ~100 tokens                                       │
└─────────────────────────────────────────────────────┘
        ↓
┌─────────────────────────────────────────────────────┐
│ 【可用技能】 (dynamic, from buildSkillsPromptSection)│
│ - music: 处理点歌...                                 │
│ - reminder: 处理提醒...                             │
│ - air-conditioner: 处理空调...                      │
│ - game: 处理游戏...                                 │
│ ~50 tokens (all 4 skills)                          │
└─────────────────────────────────────────────────────┘
        ↓
    = Full System Prompt (~150 tokens, minimal)

User Request Turn:
┌─────────────────────────────────────────────────────┐
│ LLM reads skill list, judges if load_skill needed  │
│ IF LLM calls load_skill("music"):                   │
│   ├─ Receives full SKILL.md body                   │
│   └─ ~500-1000 tokens for detailed rules           │
│                                                     │
│ ELSE: keeps context minimal                        │
│   └─ Continues without skill details               │
└─────────────────────────────────────────────────────┘

Result: Token efficiency through progressive disclosure
- No unnecessary context inflation
- Scales better with more skills
- Rules only loaded when relevant
